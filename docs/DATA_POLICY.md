# 데이터 상태와 가짜 데이터 정책

## 원칙

COSPI WTS는 숫자 데이터를 임의로 생성하지 않습니다. API 키가 없거나 제공자가 응답하지 않으면 빈 값과 상태를 표시합니다.

## 상태 정의

| 상태 | 의미 |
| --- | --- |
| `REALTIME` | 실시간 스트리밍 또는 실시간 계약 기반 데이터 |
| `NEAR_REALTIME` | REST 조회 기반 근실시간 데이터 |
| `DELAYED` | 공개 RSS, 공시, 지연 시세 등 지연 가능 데이터 |
| `API_REQUIRED` | 키, 계좌, 서비스키, 계약이 필요함 |
| `NO_DATA` | 정상 조회했지만 항목이 없음 |
| `RATE_LIMITED` | 제공자 요청 제한 |
| `ERROR` | 네트워크, 인증, 제공자 오류 |

## 화면 정책

- 가격, 금리, 환율, 수익률이 없으면 `--`로 표시합니다.
- 숫자 대신 상태 배지를 함께 보여줍니다.
- 뉴스는 공개 RSS 기반이므로 `지연 데이터`로 표시합니다.
- OpenDART는 공시 시점과 제공자 응답 기준으로 표시하며, 종목코드-기업 고유번호 매핑에 실패하면 `데이터 없음`으로 표시합니다.
- 포트폴리오 평가금액은 현재가 API가 없으면 계산하지 않습니다.
- 국내 지수와 옵션 체인은 KIS가 반환한 값만 표시하며, 확인되지 않은 지수/옵션 코드는 `데이터 없음`으로 둡니다.
- 실적/재무는 KIS가 반환한 재무제표/비율 행만 표시하며, 누락 항목은 `--`로 둡니다.

## 차트 정책

- 기간과 인터벌 변경 시 `/api/market/chart`를 다시 호출합니다.
- KIS 키가 없으면 캔들 차트를 비워두고 `API 필요` 상태를 표시합니다.
- 지표(MA, Bollinger, RSI, MACD)는 실제 캔들 배열에서만 계산합니다.

## 공시 정책

- OpenDART 기업 고유번호 목록은 공개 데이터 캐시로 `data/dart-corp-codes.json`에 저장합니다.
- 캐시가 없거나 만료되면 서버가 OpenDART에서 다시 내려받습니다.
- 공시 목록은 OpenDART 응답 그대로 지연 데이터로 표시합니다. 사업보고서, 정정공시, 소유상황보고서 등을 임의로 선별하거나 순서를 바꾸지 않습니다.

## 포트폴리오 정책

- KIS 잔고와 수동 입력 포트폴리오를 합산할 수 있습니다.
- 수동 보유 종목도 현재가 API가 없으면 평가금액과 손익을 계산하지 않습니다.
- 배당 예상 내역은 사용자가 입력하거나 추후 배당 API를 연결해야 하며 임의 추정하지 않습니다.
- 자산 추이 그래프는 포트폴리오 조회 시 저장된 실제 평가금액 스냅샷만 사용합니다. 과거 자산값을 임의로 되채우지 않습니다.
- 체결 내역은 KIS 주식일별주문체결조회 응답만 표시합니다. 계좌번호가 없거나 조회 결과가 없으면 샘플 거래를 만들지 않습니다.

## 알림 정책

- 가격 알림은 사용자 설정의 종목코드, 조건, 목표가를 저장합니다.
- 조건 평가는 KIS 현재가처럼 실제로 조회된 시세 값이 있을 때만 수행합니다.
- 현재가가 없거나 API가 필요한 종목은 알림을 임의로 발생시키지 않습니다.
- 알림은 현재 브라우저 화면 알림으로 표시하며, 외부 푸시/문자/메일 전송은 구현하지 않았습니다.

## 주문 정책

- 모의투자와 실전투자를 색상과 라벨로 분리합니다.
- 실전투자 주문은 서버와 사용자 설정이 모두 켜져야 합니다.
- 실전투자 확인 문구가 없으면 서버에서 차단합니다.
- 체결 내역 조회는 읽기 전용 API이며 주문 전송 API와 분리되어 있습니다.
