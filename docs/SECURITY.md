# 로그인과 보안 설계

## 인증

- 이메일/비밀번호 기반 로컬 계정을 제공합니다.
- 비밀번호는 `bcryptjs`로 해시합니다.
- 세션은 서명된 HTTP-only 쿠키 `cospi_session`으로 유지합니다.
- 운영 환경에서는 HTTPS 뒤에서 실행하고 `NODE_ENV=production`으로 `secure` 쿠키를 사용하세요.

## API 키 저장

- KIS, DART, Gemini 키는 서버 저장소 `data/store.json`에 AES-256-GCM으로 암호화 저장됩니다.
- 암호화 키는 `.env`의 `MASTER_KEY`에서 파생됩니다.
- `MASTER_KEY`를 잃으면 저장된 API 키를 복호화할 수 없습니다.
- API 키 원문은 클라이언트 응답에 포함하지 않습니다.
- 관심종목, 가격 알림, 수동 포트폴리오 같은 일반 설정은 암호화 대상 비밀이 아니지만 사용자별 `data/store.json`에 저장됩니다.
- 잔고와 체결 내역 API는 로그인된 사용자 세션에서만 호출하며, 비로그인 요청은 차단합니다.

## 운영 체크리스트

- `SESSION_SECRET`과 `MASTER_KEY`를 충분히 긴 난수로 설정합니다.
- `.env`와 `data/`는 Git에 커밋하지 않습니다.
- `data/dart-corp-codes.json`은 공개 기업코드 캐시지만 운영 데이터와 함께 `data/` 아래에 있으므로 배포 ZIP과 Git에는 포함하지 않습니다.
- 리버스 프록시에서 HTTPS, HSTS, request size limit, 접근 로그 보존을 설정합니다.
- 컨테이너 볼륨 `cospi-data`를 백업합니다.
- 실전투자 주문은 기본적으로 꺼져 있습니다. 서버에서 `ALLOW_LIVE_TRADING=true`를 설정하고 사용자 설정도 켜야 합니다.
- 법인/제휴 형태로 시세를 재표출할 경우 한국투자증권, 거래소, 코스콤 등과 데이터 이용 조건을 별도 확인해야 합니다.

## 토큰 발급

- KIS access token은 서버 메모리에 캐시합니다.
- 토큰은 브라우저에 저장하지 않습니다.
- 토큰 발급 실패나 만료는 `API 필요`, `오류`, `요청 제한` 상태로 표시합니다.

## 남은 보안 확장

- 운영용 DB(PostgreSQL 등)와 서버 세션 저장소(Redis 등)로 교체
- CSRF 토큰
- 사용자별 접근 로그와 주문 감사 로그
- 키 회전 및 폐기 기능
- 관리자용 실전투자 잠금 정책
